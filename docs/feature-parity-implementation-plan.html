01 · Discovery

Establish verified parity

GPT-5.6 Sol and Claude Fable independently audit the existing llm_notch implementation, publicly documented Vibe Island behavior, current Cursor/Claude Code/Codex hook APIs, and native platform restrictions. Their parity matrix, architecture, UX, and acceptance criteria must be reconciled before implementation.

02 · Integrations

Zero-manual-configuration connector manager

  • DetectFind installed agents and user/project configuration paths.
  • PreviewShow a human-reviewable diff before any modification.
  • ProtectBack up, lock, hash-check, and atomically merge configuration.
  • PreserveKeep every unrelated existing hook and configuration entry.
  • ConnectUse the bundled helper’s absolute, signed path.
  • RecoverSupport health checks, repair, disable, and exact rollback.
  • DefendReject path escapes, unsafe symlinks, and concurrent changes.
  • SimplifyRequire one onboarding confirmation, not terminal commands.
03 · Adapters

Agent-specific vertical slices

Build and test adapters for Cursor, Claude Code, Codex, and the open generic protocol. Each adapter declares version-specific capabilities for lifecycle events, permissions, questions, plan review, context jumping, process attribution, and usage information. Unsupported controls are absent rather than simulated.

04 · Parity

Interaction and workflow parity

  • Live sessionsReliable lifecycle, tool, completion, and failure events.
  • DecisionsApprove/reject only when an official acknowledged path exists.
  • QuestionsAnswer agent prompts through capability-gated adapters.
  • PlansAccessible Markdown plan review and feedback.
  • ContextJump precisely to the originating editor or terminal session.
  • AlertsNative sound and notification behavior without stealing focus.
  • UsageProvider quota and usage where reliable local sources exist.
  • HealthDetect broken integrations and offer reviewed repair.

06 · Execution

Ten-agent execution model

  1. 1 · Connector managerDetection, preview, backup, merge, repair, disable, rollback.
  2. 2 · Cursor adapterHooks, normalization, capability/version and health checks.
  3. 3 · Claude Code adapterLifecycle, permissions, plans/questions, and health.
  4. 4 · Codex + genericCurrent hooks, legacy fallback, and open SDK.
  5. 5 · Decision brokerAcknowledged decisions, answers, expiry, and failure.
  6. 6 · Context navigationSafe editor/terminal activation on each platform.
  7. 7 · Native UXOnboarding, reviewed diffs, progress, repair, and accessibility.
  8. 8 · ObservabilityUsage, sounds, alerts, metrics, and history.
  9. 9 · Release engineeringUpdater, signing, installers, and CI matrices.
  10. 10 · Integration + QAResolve interface drift, test, harden, and release.

Agents 1–4 establish and freeze contracts first. Agents 5–9 execute against those contracts. Agent 10 runs only after all implementation branches are ready. No concurrent ownership of root manifests or shared contracts is allowed.

07 · Release gate

Definition of done

  • One-click setupA fresh install connects supported agents after one reviewed confirmation.
  • LatencyA new Cursor session appears within 500 ms of an accepted event.
  • SafetyExisting hooks remain intact and repeated installs are idempotent.
  • RecoveryDisable and rollback restore the exact previous configuration.
  • Truthful actionsDecisions resolve only after vendor acknowledgement.
  • Browser gateChromium and WebKit Playwright/axe suites pass.
  • Native gatemacOS and Windows native integration tests pass.
  • Release gateSigned/notarized macOS and signed Windows packages are verified.
  • Review gateNo P0/P1 finding remains from either reviewer.

GOAL

Bring llm_notch to publicly observable feature parity with Vibe Island while making it better through:

- Native Windows and macOS support
- Zero-manual-configuration agent integrations
- Real host and per-agent resource metrics
- An open local integration protocol
- Better integration health, repair, history, privacy, and diagnostics

Do not copy Vibe Island branding, text, assets, or proprietary implementation. Use only publicly observable behavior and official vendor documentation.

CURRENT STATE

llm_notch is a Tauri 2 application with:

- Native overlay and dashboard windows
- Rust core and SQLite persistence
- Authenticated local IPC and bundled hook helper
- Host/per-agent CPU, RSS, runtime, process, and I/O metrics
- Cursor, Claude Code, Codex, and generic hook templates
- Native renderer and Playwright test mode

The major parity gap is integration installation. Hooks currently require manual installation, and connector Apply/Remove operations are deliberately disabled.

MODEL ROUTING

Use:

- GPT-5.6 Sol for systems architecture, security, protocol design, orchestration, integration decisions, and technical review.
- Claude Fable for product design, onboarding, interaction design, accessibility, native UX, copy, and visual review.
- Up to 10 Composer 2.5 Fast subagents for implementation and execution.

NON-NEGOTIABLE REQUIREMENTS

Zero-config onboarding:
- Detect installed Cursor, Claude Code, and Codex configurations.
- Present one explicit “Connect agents” confirmation.
- Show a diff before modifications.
- Preserve unrelated hooks.
- Back up existing configuration.
- Use atomic writes and file locking.
- Detect concurrent changes.
- Reject symlinks and path escapes.
- Support install, health check, repair, disable, and exact rollback.
- Use the bundled helper’s absolute path.
- Never require users to manually edit JSON or configure environment variables.

Adapter behavior:
- Use official hook APIs and retrieve current documentation.
- Detect vendor/version capabilities.
- Sessions must appear within 500 ms of accepted events.
- Support lifecycle, permission, question, plan, completion, and failure events where available.
- Only show approve/deny/answer controls when the vendor version supports a verified response path.
- Never simulate success.
- Preserve fail-open behavior so telemetry cannot block an agent.
- Redact raw prompts, command bodies, tool output, secrets, and credentials by default.

Native behavior:
- Keep separate non-activating overlay and focusable dashboard windows.
- Preserve foreground application focus.
- Support tray, global shortcut, startup, multi-monitor, scaling, and reduced motion.
- macOS should use the strongest reliable NSPanel/AppKit implementation.
- Windows should use a topmost no-activate tool window with correct DPI behavior.
- Do not claim unsupported fullscreen behavior.

Better-than-parity:
- Host and per-agent process-tree CPU, memory, runtime, process count, and I/O.
- Timestamp-accurate 15m/1h/24h history.
- Resource alerts without focus stealing.
- Integration health and repair diagnostics.
- Open, versioned generic event protocol.
- Honest attribution quality: exact/shared/heuristic/unavailable.
- Local-only operation with no cloud dependency.
- Privacy controls, data purge, bounded persistence, and audit history.
